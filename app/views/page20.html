Alors, P5 ou Processing ?

Je ne peux pas avoir un avis tranché sur cette question, alors je vous propose l'algorithme suivant ;)

Si vous êtes à l'aise avec JAVA, allez vers Processing.

Si vous êtes à l'aise avec Javascript, allez vers P5.

Si vous êtes à l'aise avec Python, essayez le projet Processing.py (que je n'ai pas testé personnellement mais qui semble prometteur).

Pour ma part j'ai un faible pour P5, car j'aime beaucoup le langage Javascript (JS) que je trouve particulièrement bien adapté pour l'expérimentation graphique et sonore.

Doté d'un modèle objet très souple, JS simplifie la manipulation de données structurées (tableaux d'objets notamment), le typage faible des variables est idéal pour le prototypage, et le fait qu'il soit reconnu nativement par tous les navigateurs est un formidable atout.
